# video-sub-md 开发日志

## 1. 项目起源

### 原始需求
用户需要一个能够批量下载 Bilibili 和 YouTube 视频字幕的工具，并希望下载完成后能自动调用 AI 生成内容摘要。

### 核心动机
- 手动复制粘贴字幕效率低下
- 多平台字幕格式不统一
- 看完视频后难以快速提取核心内容

## 3. 踩坑记录

### 坑 1：中文文件名被截断

| 项目 | 内容 |
|------|------|
| **问题** | 文件名如 `【中配】月入7.7万美元的独立开发者，每天只工作4小时？他的秘诀竟然是疯狂发布产品！.md` 被截断成 `【中配】月入7.md` |
| **根因** | `Path.with_suffix(".md")` 把中文中的 `.md` 误识别为原后缀，导致截断 |
| **解决方案** | 改用字符串拼接 `Path(str(output_path) + suffix)` 替代 `with_suffix()` |
| **涉及版本** | v0.2.0 |

### 坑 2：DeepSeek API 输出截断

| 项目 | 内容 |
|------|------|
| **问题** | AI 分析内容被截断，只输出 60-70% 就结束 |
| **根因** | `max_tokens=800` 不足以承载四层分析协议（约需 3000+ 中文字符） |
| **解决方案** | 逐步提升 max_tokens：800 → 4096 → 32768 |
| **涉及版本** | v0.2.0 → v0.3.0 |

### 坑 3：API 请求超时

| 项目 | 内容 |
|------|------|
| **问题** | DeepSeek API 调用时出现 timeout 错误 |
| **根因** | `timeout=60` 秒对于长输出不够，尤其在启用推理模式后 |
| **解决方案** | timeout 从 60 秒增加到 180 秒 |
| **涉及版本** | v0.3.0 |

### 坑 4：文件名字符非法

| 项目 | 内容 |
|------|------|
| **问题** | Windows 文件名保留字符 `\\/:*?"<>|` 导致文件无法创建 |
| **根因** | 原方案将非法字符替换为下划线，但下划线仍可能造成语义不清 |
| **解决方案** | 直接删除非法字符，而非替换 |
| **涉及版本** | v0.2.0 |

### 坑 5：DeepSeek 模型弃用

| 项目 | 内容 |
|------|------|
| **问题** | deepseek-chat 和 deepseek-reasoner 将于 2026-07-24 弃用 |
| **根因** | DeepSeek API 官方公告，旧模型名称将逐步淘汰 |
| **解决方案** | 切换至 deepseek-v4-pro / deepseek-v4-flash，并添加 thinking 参数 |
| **涉及版本** | v0.3.0 |

## 4. 设计决策

### 决策 1：循环下载模式 vs 单次执行

| 选项 | 决策 | 理由 |
|------|------|------|
| ~~每次运行只处理一批~~ | 采用循环模式 | 用户需要持续下载多个批次，频繁重启程序体验差 |
| 采用 while True 循环 | ✅ | 按 q 退出，保持程序常驻 |

### 决策 2：AI 分析协议设计

| 选项 | 决策 | 理由 |
|------|------|------|
| ~~一次性输出所有分析~~ | 采用四层协议 + 可选插件 | 用户场景多样（教学/观点/混合），单一协议难以覆盖 |
| 元认知安全锁置顶 | ✅ | 防幻觉是 AI 分析的生命线 |
| 不强制数量要求 | ✅ | 避免 AI 为凑数而编造 |

### 决策 3：文件名处理策略

| 选项 | 决策 | 理由 |
|------|------|------|
| ~~替换为下划线~~ | 直接删除非法字符 | 下划线可能与原有语义混淆 |
| 保持原始长度 | ✅ | FILENAME_MAX_LENGTH 从 80 扩展到 150 |

## 5. 实际测试数据

### API 性能对比

| 模型 | max_tokens | 实际输出 | 耗时 | 稳定性 |
|------|-----------|---------|------|--------|
| deepseek-chat | 800 | ~60% 截断 | ~15s | ❌ |
| deepseek-chat | 4096 | ~70% 截断 | ~20s | ❌ |
| deepseek-v4-flash | 32768 | 100% 完整 | ~25s | ✅ |
| deepseek-v4-pro + thinking | 32768 | 100% 完整 | ~40s | ✅ |

### 字幕处理能力

| 字幕长度 | 处理方式 | 输出完整性 |
|---------|---------|-----------|
| < 3000 字 | 单轮模式 | ✅ 100% |
| 3000-8000 字 | 多轮模式（推荐） | ✅ 100% |
| > 8000 字 | 分段 + 跨段关联 | ✅ 100% |
